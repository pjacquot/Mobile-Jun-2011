Assignment: Basic Design Principles of a Mobile Web page

Assignment 2A

During this assignment we will make the design of a web page about Beethoven's Ode to Joy mobile friendly. On various mobile browsers, the page is zoomed out and required zoom in then double scrolling to be read.

Web page original version

Step A: Remove double scrolling

The following modifications are applied at this step:

  • Add the <meta name="viewport" content="width=device-width,initial-scale=1.0"/> tag.
  • Resize image beethoven.jpg to 120x150 and cropp + resize beethoven-background.jpg to 120x18.
  • Changed the sidebar from left float to block, making the page linear with a liquid layout.

To make the table more usable on a mobile device, I've applied these adaptations:

  • The "Symphony" column header is renamed "Symphony nÂ°" and the word Symphony is removed from following rows.
  • The the word "Op." is removed from the rows following the header "Op.".
  • Padding of each cell is reduced
  • Border tables collapse in CSS (border-collapse: collapse;) so we gain separation space between cells.
Web page after step A

Step B: Revamp menu

Some elements of the menu are more useful than others. I picked the three items "Lyric Repository's Home", "More information on this artist" and "Site Map" as the most useful links. This arbitrary choice should ideally be validated and refined through user research (UX discipline).

The other menu items are moved to the bottom of the page.

I also made top menu item shorter and therefore more scannable by replacing "Lyric Repository's Home" by "Home" and "More information on this artist" by "Beethoven".

I applied the appropriate style to make the menu items as one line buttons, like I did for the week 1 assignment.

The link "lyrics of Fidelio" has been removed for two reasons:

  • It is supposed to be accessible through the category page pointed by "Other lyrics from this artist"
  • The number of other lyrics can vary unpredictably, and may become potentially huge.

I have also applied a clear:both; to the <blockquote> tag to ensure that it has the available width as it is better to avoid additional line breaks because the lyrics are in verse.

Web page after step B

Step C: Adapt images

I converted all images to GIF format, retaining the same width and height. Me might as well have considered JPEG as image format for beethoven.jpg and beethoven-background.jpg because these pictures are photographic (e.g. high color). For example, the size of beethoven.jpg file has grown from 3.5KB (JPEG version) to 13.5KB (GIF version).

Decorative and repeating music.gif icon is removed from HTML and is now set by this CSS declaration:

tbody th {
  padding-left: 22px;
  background-image: url("music.gif");
  background-position: left;
  background-repeat: no-repeat;
  background-color: white;
}
Web page after step C

Step D: Adapt styles to different viewports

For the adaptative version of the page, I created these files:

beethoven-background-high.jpg
Version of the background for the higher resolution (width > 800px)
beethoven-background-medium.jpg
Version of the background for the medium resolution (600px < width < 800px)
style-base.css
Base version of the stylesheet, always applied
style-medium.css
Medium version of the stylesheet, applied in addition to the base stylesheet (600px < width < 800px)
style-high.css
High version of the stylesheet, applied in addition to the base stylesheet (600px < width < 800px)

I added the following style directives to the header of the web page. Nothing else changed in sample.html.

  <meta name="viewport" content="width=device-width,initial-scale=1.0" />
  <link rel="stylesheet" type="text/css" href="style-base.css" />
  <link rel="stylesheet" type="text/css" href="style-medium.css" media="only all and (min-width: 600px) and (max-width: 800px)" />
  <link rel="stylesheet" type="text/css" href="style-high.css" media="only all and (min-width: 800px)" />
  <!--[if lt IE 9 & !IEMobile]>
    <link rel="stylesheet" type="text/css" href="style-high.css" />
  <![endif]-->

I kept the style-base.css the same as the version in step C.

The following modifications are applied for the medium version (600 to 800px) in style-medium.css:

  • Make use of beethoven-background-medium.jpg
  • Removed table's table-collapse
  • Removed blockquote's clear: both

The following modifications are applied for the high version (> 800px) in style-high.css:

  • Make use of beethoven-background-medium.jpg
  • Removed table's table-collapse
  • Removed blockquote's clear: both
